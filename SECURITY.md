# Security Policy

## Reporting a Vulnerability

We take the security of Brale seriously. If you discover a security vulnerability, please report it to us as quickly as possible.

**Please DO NOT open a public GitHub issue.**

Instead, please send an email to `rzry36008@gmail.com` detailing the vulnerability. We kindly ask you to encrypt sensitive information using PGP/GPG if possible (provide your public key if you wish to receive an encrypted response).

Please include as much detail as possible in your report, including:
- A clear description of the vulnerability.
- Steps to reproduce the vulnerability.
- Proof-of-concept code or exploits.
- The version of Brale affected.

We will acknowledge your report within 2 business days and provide a more detailed response within 7 business days. We will keep you informed of the remediation process.

Thank you for helping us keep Brale secure.
